```
        │

        │
        ▼
 [output/shade_map.html · scenarios_map.html · rationales.json
  cv_buildings_overlay.png · cv_streetview_overlay.png]
```

## 6. 기술적 하이라이트

### 6.1 다변수 최적화 (채권 트레이더 방법론 이식 + CV 통합)
```
Score = 0.25·popdens + 0.25·lst + 0.20·vuln
      − 0.15·shade − 0.05·natural   (CV-A 활성)
      + 0.15·streetview_deficit     (CV-B 활성, NEW)
          (모든 피처 MinMax 정규화 후 가중합)
```

### 6.2 시간대 가중 스코어링
- 9~18시 10개 시간대별 독립 필드 생성
- HOUR_WEIGHTS 로 오후 1~3시 피크 강조 (1.0 가중)
- 역세권(출퇴근)과 시장·상업(오후) 프로파일 분리

### 6.3 보행로·횡단보도 강제 제약
- 3,672 격자 → **23 격자** (0.6%)로 축소
- 강변·건물 위 등 설치 불가능 지점 원천 제외
- 실제 정책 결정자가 바로 활용 가능한 입지만 남김

### 6.4 시나리오 민감도 분석
- 동일 피처 × 4개 가중치 프리셋 → 강건 입지 식별
- 정책 관점 불변 입지 (robust location) 3곳 확인

### 6.5 LLM 자동 근거 생성
- TOP 후보지 메타데이터 → GPT API → 정책 결정자용 자연어 설치 근거
- (API 키 없으면 룰베이스 템플릿 fallback)

### 6.6 자연 그늘 시뮬레이션 + CV-A 진화 (완료)
- **CV-A**: V-World WMTS z15 항공사진 48타일 합성 → Mobile-SAM zero-shot
  → 면적·종횡비·밝기 필터 → **실측 건물 30동** 자동 추출
  → 동작구 주요 거점(노량진·사당·이수·상도로·장승배기·흑석동) 모두 커버
- 그림자 polygon = 건물 footprint ∪ 태양반대방향 평행이동의 convex hull
  (서울 7월 말 오후 3시: 고도 52° / 방위 252°)
- 격자별 자연그늘 커버 비율 [0,1] → `natural` 음수 가중 페널티
- 산출물: `output/cv_buildings_overlay.png` (위성 + SAM mask 시각화)

### 6.7 거리뷰 컴퓨터 비전 (CV-B, NEW)
- **Mapillary**: 동작구 BBOX를 4×4 셀로 분할 검색 → **1,249장** 거리뷰 발견
- 19개 후보 격자에 nearest 매핑 → **HuggingFace SegFormer-b0**
  (`nvidia/segformer-b0-finetuned-cityscapes-1024-1024`) → 19 클래스 segmentation
- 핵심 클래스: building, vegetation, road, sidewalk, sky
- 그늘 결핍 지수: `shade_deficit = (road + sidewalk) × (1 − building − vegetation)`
- TOP10 평균 deficit **0.127** — 새 피처가 점수에 의미 있게 기여
- 산출물: `output/cv_streetview_overlay.png` (segmentation 컬러맵 시각화)

### 6.8 인터랙티브 대시보드 (P3 완료)

- 사이드바: 시나리오 프리셋 + 5개 가중치 슬라이더 (popdens·lst·vuln·shade·natural)
